## Resumo

Introdução ao conceito de paradigmas de programação como formas de pensar e estruturar soluções computacionais — independentes de linguagens específicas. A aula contextualiza por que estudar paradigmas é relevante para a carreira em desenvolvimento de software e apresenta uma linha do tempo das linguagens de programação desde os anos 1940.

---

## Pontos-chave

- **Paradigma ≠ linguagem**: paradigmas definem *como* programar; linguagens de programação os implementam. Uma linguagem pode suportar múltiplos paradigmas.
- **Por que estudar paradigmas**: o mercado evolui; compreender paradigmas permite absorver novas linguagens e tecnologias mais rapidamente e tomar melhores decisões em projetos.
- **Pré-requisitos práticos**: lógica de programação e interpretação de algoritmos são suficientes; conhecer ao menos uma linguagem ajuda na compreensão de exemplos.
- **Paradigmas mencionados**: [Programação Estruturada](../conceitos/paradigma-estruturado.md), [Orientação a Objetos](../conceitos/paradigma-oo.md), [Programação Funcional](../conceitos/paradigma-funcional.md), Programação Lógica, Programação Orientada a Aspectos, Programação Declarativa.

## Linha do Tempo das Linguagens (extraída da aula)

| Década | Linguagens / Eventos notáveis |
|--------|-------------------------------|
| 1940s | Plankalkül (Konrad Zuse), Short Code (UNIVAC I) |
| 1950–60s | FORTRAN, Lisp, ALGOL, COBOL |
| 1970s | BASIC, Pascal, C, Scheme, ML |
| 1980s | C++, Smalltalk, Perl, Haskell |
| 1990s | Python, Java, JavaScript, PHP, Ruby, Visual Basic |
| 2000s+ | C# (evoluções), Clojure, Go (Google), Swift (Apple), Hack (Meta) |

**Destaques históricos:**
- **COBOL**: popularizado pela capacidade de manipulação de bancos de dados; ainda em uso em sistemas legados em mainframes
- **C**: linguagem de propósito geral tão bem estruturada que serviu de base para C++, C#, Objective-C
- **Python**: linguagem dos anos 1990 que despontou recentemente como ideal para IA/ML, ultrapassando as demais em adoção
